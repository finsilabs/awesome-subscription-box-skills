# Homepage Review Before Paid Traffic Launch

## Problem/Feature Description

SnackCraft is a monthly artisan snack subscription box with three tiers. They are two weeks away from pushing their first paid social budget against their Shopify homepage and have asked for a thorough review of the page before they spend any money. The founder is worried they will be paying to send traffic to a page that leaks conversions.

A team member has already built the homepage on Horizon. The live URL is `https://www.snackcraft.com`. Please review it as a potential subscriber would see it, paying particular attention to anything that might stop a visitor from converting or create confusion about what the product actually is.

The review should be practical and immediately actionable — the development team can only address a limited number of issues before the launch date, so the most critical problems need to be clearly prioritized. For the highest-priority issues, the team needs specific replacement copy or code they can implement directly, not just a description of what's wrong.

## Output Specification

Produce `homepage-review.md` in the current working directory.

The review should clearly separate issues by urgency, provide an overall verdict on the page's readiness, and include scores for each key dimension of the page. For each of the most critical issues identified, include the specific fix — not just a description of the problem.
