# Complete Brand Vault — Graze

## Problem/Feature Description

A marketing operations team is standardizing how their agency documents client brands. They have a spec for what every brand vault must contain and how each file should be formatted — the spec exists because downstream tools that read these vaults are strict about field names, date formats, and principle structure. A vault with missing fields or inconsistent formats breaks the pipeline.

Graze (https://www.graze.com) is a snack subscription brand and a new addition to the agency's client roster. The ops team needs the Graze vault built to spec from scratch — every file, correct formats, no shortcuts. This is also a good test of the vault tooling because Graze has a fairly distinct personality (quirky, health-conscious, British-inflected) that will require real extraction work rather than generic copy.

The ops team needs this vault built to spec from scratch — every file, correct formats, no shortcuts. They want this vault done right the first time so the downstream tools pick it up cleanly.

## Output Specification

Research the Graze website (homepage and at least one additional page) and produce a complete brand vault with all required files. Pay careful attention to the required format for each file.

The vault should include all standard vault files. The `examples/` subfolder with source quotes is required — include at least 5 verbatim quotes from the site. All files should follow the standard format conventions for their type.
